Problem: Knots vs Links¶

In topology one draws knots/links in the 3-dimensional euclidean space by projecting them into a 2-dimensional plane and marking crossings accordingly. A knot is a single piece of string, a link is multiple. (see image for explanation) For people it's very easy to tell apart knots from links. But algorithmically it turns out to be painful.

Now we need to solve a problem: we want to travel around the graph. For that we need two functions.

  • to jump over the crossing point, find the nearest edge point
  • in a component, find the other edge point

Here we find the closest points to each edge point

Note that this is just a permutation and a pairing

B=np.zeros((A.shape))
def closestPt(pt,Set):
  index=0
  minimal=np.inf
  for i in range(0,len(Set)):
    p=(Set[i][0]-pt[0])**2+(Set[i][1]-pt[1])**2
    if p<minimal and p>0:
      index=i
      minimal=p
  return Set[index]

for i in range(len(A)):
  B[i]=closestPt(A[i],A)

def notIn(a,Set):
  for i in range(len(Set)):
    if Set[i]==a:
      return False

  return True
def otherOneInSquare(current, notThese):
  for a in range(-1,2):
    for b in range(-1,2):
      #print(skeleton[current[0]+a][current[1]+b])
      if (skeleton[current[0]+a][current[1]+b]==True) and (notIn([current[0]+a,current[1]+b], notThese)):
        return [current[0]+a,current[1]+b]

def findOtherEnd(start):
  lastPt=start
  currentPt = otherOneInSquare(lastPt, [lastPt])

  while (notIn(currentPt,list(A))):
    temp=currentPt
    currentPt = otherOneInSquare(currentPt, [currentPt,lastPt])
    lastPt=temp
  return currentPt

Now we can algorithmically check if this is a knot: start at any point. Go to the other end of the edge, then jump and repeat. If we get to the starting point without going to every point, then this is a link. else a knot.
